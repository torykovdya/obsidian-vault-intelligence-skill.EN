# Contributing

Thanks for your interest in the project. Here's how you can help.

---

## What We Need Most

### 1. New opportunity patterns
The file `skill/references/opportunity-patterns.md` contains patterns for finding opportunities in a vault. If you've spotted a pattern in your own notes that led to a real result — an article published, a product shipped, an automation built — that's a valuable contribution.

Format:
```markdown
### Pattern N: Name
**Signal in vault:** what you saw in the notes
**Opportunity type:** article / product / automation
**Example:** a concrete case (can be anonymized)
```

### 2. Vault structures for specific niches
The current structures in `skill/references/vault-structures.md` are generic. We need structures tailored to specific profiles: AI educator, automation builder, indie hacker, AI researcher, developer advocate.

### 3. Real analysis examples
Anonymized Vault Intelligence analysis outputs (with your permission) help others understand what to expect. Add them to `examples/`.

### 4. Domain-specific taxonomies
The `ai-creator-taxonomy.md` is built around AI. If you adapt it for another domain (design, fintech, biotech, creative writing), add it as `references/[domain]-taxonomy.md`.

---

## How to Submit a PR

1. Fork the repository
2. Create a branch: `git checkout -b feature/my-improvement`
3. Make your changes
4. Verify that Markdown files render correctly in Obsidian
5. Open a PR with a clear description: what you changed and why

---

## Principles All Contributions Should Follow

- **Zettelkasten-compatible**: changes should not contradict Luhmann / Ahrens principles
- **AI Creator focus**: taxonomies and patterns should be relevant to AI enthusiasts and builders
- **Actionable over theoretical**: every addition should produce a practical, usable result
- **Examples are required**: every new pattern or template must include a usage example
- **Plain language**: write as if explaining to a smart person who is new to the concept
